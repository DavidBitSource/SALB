SALB

The SALB system is a frontend framework for speech synthesis using HMM based voice models built by HTS (http://hts.sp.nitech.ac.jp/). We also provide a free voice model of an Austrian German, male, professional speaker (see https://github.com/m-toman/SALB/releases)

Facts & features:

  • SAPI5: Distribute HTS voice models as Microsoft Windows system voice. An NSIS (http://nsis.sourceforge.net/) script is provided to create an installer for your system voice (x86 and x64, starting with Windows Vista).
  • Text analysis for English voice models by flite (http://www.festvox.org/flite/).
  • Synthesis by hts_engine (http://hts-engine.sourceforge.net/).
  • Text preprocessing for new languages can be added in C++, Lexicon and Letter-To-Sound (LTS) rules can be provided in Festival (http://www.cstr.ed.ac.uk/projects/festival/) format.
  • Example text preprocessing as well as lexicon and LTS rules are provided for (Austrian) German.
  • A free Austrian German voice model of a male professional speaker is included, a voice model for a female professional speaker in Scottish English will be added soon.
  • Visual Studio project file for SAPI5 build.
  • CMake build files.
  • Simple command line interface for generating sound samples given text and a model file.
  • C++(11) API to embed in other applications.
  • Compiles on MS Windows, Linux, Android, iOS.

F.A.Q.

How can I create a new voice model?

An easy way is to use the demo packages on http://hts.sp.nitech.ac.jp/. This produces a .htsvoice file that can be used with the SALB system.

Why not use flite+hts_engine?

Cases to consider using the SALB system instead of flite+hts_engine include:

  • You need a SAPI5 interface.
  • You want to add a new language* and/or use plain text Festival lexica and LTS rules.
  • You prefer the C++ interface.

*If you really want to do this in flite, take a look our description on adding a new language to flite here: http://sourceforge.net/p/at-flite/wiki/AddingNewLanguage/

How can I contribute?

We would love to include language bindings (e.g. Python), further interfaces and more languages. Of course, bug fixes, improvements etc. are also always welcome.
