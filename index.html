SALB

The SALB system is a frontend framework for speech synthesis using HMM based voice models built by HTS (http://hts.sp.nitech.ac.jp/). We also provide a free voice model of an Austrian German, male, professional speaker (see https://github.com/m-toman/SALB/releases)

Facts & features:

  • Distribute HTS voice models as Microsoft Windows system voice via SAPI5. An NSIS (http://nsis.sourceforge.net/) script is provided to create installers (x86 and x64, starting with Windows Vista).
  • Text analysis for English voice models by flite (http://www.festvox.org/flite/).
  • Parameter and waveform generation by hts_engine (http://hts-engine.sourceforge.net/).
  • Add new languages by text preprocessing in C++ and by providing lexicon and Letter-To-Sound (LTS) rules in Festival (http://www.cstr.ed.ac.uk/projects/festival/) format.
  • Example text preprocessing as well as lexicon and LTS rules are provided for (Austrian) German.
  • A free Austrian German voice model of a male professional speaker is included, a voice model for a female professional speaker in Scottish English will be added soon.
  • Visual Studio 2012 solution file for the SAPI5 build included.
  • CMake build files for everything else.
  • Simple command line interface for generating sound samples.
  • C++(11) API to embed in other applications.
  • Successfully compiled on Windows (>= Vista), Linux, Android, iOS.

F.A.Q.

How can I create a new voice model?

An easy way is to use the demo packages on http://hts.sp.nitech.ac.jp/. These produce htsvoice-files that can be used with the SALB system.

Why not use flite+hts_engine instead?

The SALB system is another layer on top of flite and hts_engine (see https://github.com/m-toman/SALB/blob/master/doc/architecture.svg). You might want this additional layer if:

  • you want the SAPI5 interface.
  • you would like to add a new language and/or use plain text Festival lexica and LTS rules*.
  • you prefer the C++ interface.
  • you would like to use the provided German voice model.

*Of course this is also possible in flite but a bit trickier. If you would really like to do that, take a look our description on adding a new language to flite here: http://sourceforge.net/p/at-flite/wiki/AddingNewLanguage/.

How can I contribute?

We would love to include language bindings (e.g. Python), further interfaces and more languages. Of course, bug fixes, improvements etc. are also always welcome.
